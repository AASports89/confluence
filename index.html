🔍 Search

Encounters 🔍 Search

Features three of the following complementing data-oriented tabs:

1 - Encounter:
  • Helpful in locating virtually any billing encounter recorded in the system
  • Data filters & fields are geared more towards the various data entities typically present in billing encounters
  • Users are able to Payment Source Info to locate encounters
    • 2 - Workflow:
  • Utilized to locate encounters going to the various workflows present in the platform
  • Features a unique Queue data filter to narrow results based on a workflow queue
  • The ability to isolate search results further based on Flags & Status
    • 3 - LIS:
  • Important to validate the accuracy based on the ordered tests & other provider controlled logistics
  • Contains numerous unqiue exclusion filters to narrow search results even more
  • Has a powerful combination of drop-down data filters connected to lab testing, results & CPT codes
    • 🗂️ Demographics
      Performing An Address Verification in the Demographics Tab on an address field:
  • “Click” on the WP icon or ℹ️ icon to initiate the address verification function
  • “Click” on Accept New Address - Note: the ✅ icon in the right-hand corner of the address data field:
  • “Real-time” indication appears as the ✅ icon confirming address verification has been completed & validated

    🗂️ Insurance Review

    Insurance Review Workflow

    Guide: Primary Claim Resubmission

  • Select the Account corresponding to the claim in the Charges section via “right-clicking” & selecting the 🔄 Re-submit Primary Claim option
  • A separate 🖥️ Create Insurance Claim window appears displaying data fields corresponding to the claim re-submission:
  • CLM #
  • Payer
  • Sequence
  • Next Payer
  • Type
  • Frequency Code
  • Indicator Code
  • Transaction Code
  • “Click” on the 🔍 Preview CMS 1500 button to view indicated form
  • “Click” on the 🔍 Preview 837 button to view indicated transmission data
  • After “clicking” on the ✅ F1: Confirm button new data row appears w/in the Timeline Tab including the following:
  • Post Date
  • Event
  • Status
  • Payer
  • CLM / INV #
  • Description
  • Balance
  • Display processed data in the Timeline Tab via “checking-off” the Show Effective ⬜ box
  • Current re-submission linked data rows NO longer appear in the Timeline Tab
