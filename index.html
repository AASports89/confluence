Workflow Guide: 📂 Encounters

🔍 Search
🔍 Search - Encounter Tab

The Encounter Tab is designed to assist users to isolate specific groups and individual billing encounters:

  • Using the variety of pre-configured data filters and input fields.
  • Filters and fields can be used to isolate what type of encounter data will be shown, grouped by, and include a template if applicable.
  • Additionally, date of service (DOS) combined with Client Account, Provider, Diagnoses, and custom pre-configured workflow-related tags are available to further narrow search results.
  • Inputted fields are applicable to drop-down filters resulting in a quicker route to selecting the accurate data filter choice.
  • Lastly, search populated encounters can be accessed via "clicking" on the 📝 Actions button or simply "double-clicking" on the search result.
    • 🔍 Search - Workflow Tab

    The Workflow Tab is geared to isolating specific groups and individual workflow-related billing encounters:

  • Consists of date-centric, user-related, and workflow queue specific date filters.
  • The filters linked to showing, grouping, and including a template are also available.
  • Queue, Flag, and Status are the 3-unique filters w/ the ability to isolate encounter search relates based on workflow related tasks.
    • 🔍 Search - LIS Tab

    The LIS Tab is designed to assist users to isolate specific groups and individual workflow related billing encounters:

  • Consists of date-centric, user-related, and workflow queue specific date filters.
  • The filters linked to showing, grouping, and including a template are also available.
  • Queue, Flag, and Status are the 3-unique filters w/ the ability to isolate encounter search relates based on workflow related tasks.
    • 🗂️ Demographics
      Performing An Address Verification in the Demographics Tab on an address field:
  • “Click” on the WP icon or ℹ️ icon to initiate the address verification function
  • “Click” on Accept New Address - Note: the ✅ icon in the right-hand corner of the address data field:
  • “Real-time” indication appears as the ✅ icon confirming address verification has been completed & validated

    🗂️ Insurance Review

    Walkthrough: Primary Claim Resubmission

  • Select the Account corresponding to the claim in the Charges section via “right-clicking” & selecting the 🔄 Re-submit Primary Claim option
  • A separate 🖥️ Create Insurance Claim window appears displaying data fields corresponding to the claim re-submission:
  • CLM #
  • Payer
  • Sequence
  • Next Payer
  • Type
  • Frequency Code
  • Indicator Code
  • Transaction Code
  • “Click” on the 🔍 Preview CMS 1500 button to view indicated form
  • “Click” on the 🔍 Preview 837 button to view indicated transmission data
  • After “clicking” on the ✅ F1: Confirm button new data row appears w/in the Timeline Tab including the following:
  • Post Date
  • Event
  • Status
  • Payer
  • CLM / INV #
  • Description
  • Balance
  • Display processed data in the Timeline Tab via “checking-off” the Show Effective ⬜ box
  • Current re-submission linked data rows NO longer appear in the Timeline Tab
